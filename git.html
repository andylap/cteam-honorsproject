Using Git

Overview

This section will briefly go over essential git commands you can use to interface with tools such as BitBucket, Source Forge, and most popularly, GitHub. We will discuss basic command line instructions to just get started and move on to slightly more complex commands that will be needed when working in teams, especially for the ECE 39595 final project!

Why use Git?

Simply, it makes collaboration easier. Git will track whwat you have added to source files, what you have deleted, allow you to revert to previous version, and so much more. In projects with thousands of contributors, like open source projects, git is essential in making sure different people don't cause conflicts or break the current system. With engineers, it's all about efficiency, and git delivers exactly that when it comes to project management.

Git Structure

There's 4 main components to git architecture: Working directory, Staging area, Local repository, Remote repository

  • All the adding/editing/deleting you will be doing to your source files will be done in the working directory
  • Anything you want to save for your next version (A.K.A commit) of your project is in the staging area. Kind of like a rough draft.
  • Where you make local changes, probably your computer, is your local repository. Anything "committed" will be here.
  • The project is hosted "remotely" on some server in some far away place. All team members have access to this remote repository.
