This section will give some background information regarding compilers and a brief overview of how to choose and install a compiler/Integrated Development Environment (IDE) for your programming needs.

Compiled vs. Interpreted Languages

First, a little background information.

There are two typical ways that code is executed, and that is dependent on if the language is compiled or interpreted (although there is some grey area).

Code written in compiled languages are executed directly on a computer’s processer where a compiler turns the code into machine language that the processor can use. Then, at runtime the machine language is used to execute your program.

Interpreted languages, on the other hand, are programming languages that are not machine code before runtime (no separate compilation step). Therefore, the code is translated to machine language at runtime when the code is being executed, potentially slowing down the process.

Java is a little different than C++, as it is a language that is neither compiled nor interpreted. In C or C++, the executable files generated after compilation can only be run on that specific operating system (a C++ program compiled on a Mac can only run on a Mac).

Java, however, takes a different approach. Java designers reorganized the compile-link-execute sequence used in C/C++, and developed low level means of performing that sequence with platform independence.

While a C/C++ compiler creates an object file, the Java compiler (often referred to as the Java Virtual Machine) creates a bytecode file, which is an object file for the virtual machine.

The steps for compiling and running a program in Java work like this:

  1. Write in IDE
  2. JVM
  3. execute with java on platform

Is Python interpreted or compiled?

Answer? It depends. Python is officially classified as an interpreted language, as it generates bytecode (machine langauge) from your source code and and executes the bytecode in one step.

The process of compiling and linking in python is similar to Java. Source code is saved as a .py file, and then that is compiled to bytecode like Java (platform independent). Bytecode is stored in .pyc files that can be run by the PVM (Python Virtual Machine).

If you're curious about where the line is between interpreted and compiled langauges, the Python documenation explains it like this:

"Python is an interpreted language, as opposed to a compiled one, though the distinction can be blurry because of the presence of the bytecode compiler. This means that source files can be run directly without explicitly creating an executable which is then run."
